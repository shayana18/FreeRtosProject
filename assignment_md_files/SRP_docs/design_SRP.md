This SRP implementation extends the existing EDF scheduler so shared resources are handled with the Stack Resource Policy instead of falling back to FreeRTOS's fixed-priority resource behavior. In EDF + SRP mode, the kernel keeps a dedicated SRP ready list ordered by absolute deadline, a registry of SRP tasks, and extra per-task metadata for preemption level, declared resource claims, currently held resources, and requested stack depth. `xTaskCreate()` is overloaded again in this mode so task creation validates the EDF timing parameters and SRP claim table together, derives the task's preemption level from its relative deadline, and inserts the task into the SRP-specific kernel data structures.

At runtime, job dispatch still follows EDF order, but a ready task is only allowed to run if its preemption level is above the current system ceiling or if it already holds resources and can safely resume. The system ceiling is recomputed whenever SRP resources are acquired or released through the SRP semaphore wrappers, and held resources are force-released if a task is deleted. On top of the protocol logic, the kernel optionally supports shared run-time stacks for SRP tasks: tasks at the same preemption level share a kernel-owned stack region sized by the largest task at that level, and `vTaskGetSRPSharedStackUsage()` reports shared-stack bytes versus private-stack bytes for comparison. The current layout is intentionally static, so SRP tasks are expected to be created before the scheduler starts and the shared-stack regions are not reorganized once execution begins.
